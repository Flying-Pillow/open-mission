# Replace Sandcastle Dependency With Mission-Owned Agent Runners

Objective: remove the `@ai-hero/sandcastle` runtime dependency and fully delete the current adapter-plus-runner stack in favor of lean Mission-owned runner implementations for `claude-code`, `codex`, `opencode`, and `pi`.

Context: keep the Mission-owned PTY transport, session lifecycle, signal routing, signal policy, MCP server, and Airport projection work already established in earlier slices. Treat the Sandcastle repository as a behavioral reference for command construction and stream parsing only. This slice is intentionally KISS: do not keep Sandcastle as a production dependency and do not preserve the `AgentProviderAdapter` / `SandcastleAgentRunner` layering under any name.

Allowed files: `packages/core/package.json`, `pnpm-lock.yaml`, `packages/core/src/daemon/runtime/agent/runtimes/AgentRuntimeFactory.ts`, `packages/core/src/daemon/runtime/agent/runtimes/CopilotCliAgentRunner.ts`, `packages/core/src/daemon/runtime/agent/runtimes/CopilotCliAgentRunner.test.ts`, `packages/core/src/daemon/runtime/agent/runtimes/PiAgentRunner.ts`, `packages/core/src/daemon/runtime/agent/runtimes/PiAgentRunner.test.ts`, new or existing Mission-owned runner or helper files under `packages/core/src/daemon/runtime/agent/runtimes/**`, and deletion of `packages/core/src/daemon/runtime/agent/providers/AgentProviderAdapter.ts`, `packages/core/src/daemon/runtime/agent/providers/AgentProviderAdapter.test.ts`, `packages/core/src/daemon/runtime/agent/runtimes/SandcastleAgentRunner.ts`, and `packages/core/src/daemon/runtime/agent/runtimes/SandcastleAgentRunner.test.ts`.

Forbidden files: `apps/airport/**` outside existing provider-neutral interaction-mode projection work; `packages/core/src/daemon/runtime/agent/TerminalAgentTransport.ts`; `packages/core/src/daemon/runtime/agent/signals/**` except narrowly required integration adjustments; `packages/core/src/daemon/runtime/agent/mcp/**` except env or launch-context wiring that is unavoidable for the new runner shape; `docs/adr/**`; any Sandcastle sandbox/worktree/orchestration integration files; workflow gate or verification code; tracked files containing per-session MCP credentials.

Expected change: remove `@ai-hero/sandcastle` from Mission; delete `AgentProviderAdapter` and `SandcastleAgentRunner` completely; inline only the command-launch and output-parsing behavior Mission truly needs; introduce explicit Mission-owned PTY-backed runner implementations, or at most one very small local shared helper, for `claude-code`, `codex`, `opencode`, and `pi`; keep runner registration explicit in `AgentRuntimeFactory`; preserve provider-specific metadata handling for model, effort, permission bypass, resume, and usage/session parsing where supported; refactor `PiAgentRunner` onto the new local shape; and align `CopilotCliAgentRunner` with the leaner PTY runner conventions where that reduces duplication without introducing a new generic provider abstraction.

Compatibility policy: preserve the existing Mission-owned PTY launch path, session lifecycle, signal observation routing, and runner ids; keep unsupported capabilities honest; do not preserve `AgentProviderAdapter`, `SandcastleAgentRunner`, or any renamed replacement for those abstractions. The desired end state is direct Mission-owned runners with no extra provider-adapter layer.

Validation gate: `pnpm --filter @flying-pillow/mission-core check`, `pnpm --filter @flying-pillow/mission-core test`, `pnpm --filter @flying-pillow/mission-core build`.

# Simplify Agent MCP Integration

Objective: keep Mission-owned MCP signaling but remove per-agent MCP config materialization so agent coders are instructed to use the Mission MCP server instead of relying on runner-specific config-file mutation.

Context: preserve the local Mission MCP signal server, session registry, signal port, signal policy, and protocol-marker fallback established in earlier slices. The simplification target is the agent-coder-specific provisioning layer under `packages/core/src/daemon/runtime/agent/mcp/**` that currently writes or patches runner-specific config files for Claude Code, Codex, OpenCode, and Pi.

Allowed files: `packages/core/src/daemon/runtime/agent/mcp/**`, `packages/core/src/daemon/runtime/agent/runtimes/**` only as needed to pass launch env or launch instructions through the new leaner path, `.agents/skills/mission-agent-runtime-protocol/SKILL.md`, and existing provider-neutral launch-instruction or signal-boundary files only as needed to reflect instruction-guided MCP access.

Forbidden files: `apps/airport/**` outside provider-neutral interaction-mode projection work; `packages/core/src/daemon/runtime/agent/TerminalAgentTransport.ts`; `docs/adr/**`; any remote MCP service or hosted endpoint; tracked files containing per-session MCP credentials; any new per-agent config materializer, config-patching helper, or universal `.agents/mcp.json` assumption.

Expected change: remove runner-specific MCP config materializers and the maintenance-heavy provisioning path that mutates coder-specific config files; keep one Mission-owned MCP server and session registration boundary; expose the session-scoped MCP endpoint and identity through launch env and explicit runtime instructions; tell agents how to use the local Mission MCP server when their tooling supports it; preserve strict fallback markers when MCP is unavailable or unsupported; and keep degraded or unavailable MCP states honest without pretending Mission has validated first-class MCP wiring for every coder.

Compatibility policy: preserve Mission-owned signal ingestion and policy authority, but do not preserve per-runner `.mcp.json`, `.codex/config.toml`, `opencode.json`, or similar config-writing paths once instruction-guided MCP access is in place.

Validation gate: `pnpm --filter @flying-pillow/mission-core check`, `pnpm --filter @flying-pillow/mission-core test`, `pnpm --filter @flying-pillow/mission-core build`.

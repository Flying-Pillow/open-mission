Start a Mission when the repository is set up and the work has a clear intent.

Mission can begin from a new brief or an existing GitHub issue resolved through the repository's platform adapter.

## What Happens

1. The daemon creates a Mission branch ref and Mission worktree.
2. Mission records the initial Mission dossier.
3. Workflow law generates the first stage, artifacts, and tasks.
4. Airport shows the Running Mission instance, available commands, artifacts, and agent sessions.

Read the current artifact, decide which command is legal, launch or pause agent work, inspect evidence, and advance only when the gate is satisfied.

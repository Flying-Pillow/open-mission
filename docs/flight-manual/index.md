# Flight Manual

The flight manual is the onboarding layer for the documentation set.

Use it to understand how the Mission docs are organized and where the current authoritative material lives.

## Why There Is A Missions Section

Mission is still being built.

That means we need a temporary separation between:

- documentation of the intended finished system
- active design and delivery artifacts required to build that system

The `missions` area is where the current work-in-progress specification set lives until the product can publish its own stable delivered documentation.

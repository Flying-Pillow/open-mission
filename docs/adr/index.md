ADRs are the durable record of Mission architecture decisions.

Read them when a change touches ownership, naming, state, runtime behavior, repository setup, Airport surfaces, or compatibility policy.

Each ADR carries frontmatter so the docs site and architecture skills can treat decisions as structured records.
